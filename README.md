# java-lotto-precourse

## 개요
간단한 로또 발매기 미션입니다. 객체지향 설계와 단위 테스트 작성 연습을 목표로 합니다.

## 학습 목표
- 관련 함수를 클래스로 묶어 객체들이 협력하도록 구현
- 클래스와 함수 단위의 단위 테스트 작성으로 안정성 확보
- 2주 차 공통 피드백을 반영한 리팩터링

## 구현할 기능 목록

- 구매 금액 입력 받기
  - 1,000원 단위인지 검증 (나누어떨어지지 않으면 예외)
- 로또 발행
  - 로또 한 장에 중복 없는 6개 숫자(1~45)
  - `camp.nextstep.edu.missionutils.Randoms.pickUniqueNumbersInRange(1, 45, 6)` 사용
  - 구매 금액에 맞춰 N장 발행, 각 로또 번호는 오름차순으로 정렬하여 출력
- 당첨 번호 입력 받기
  - 쉼표로 구분된 6개 번호 검증 (범위, 중복 검사)
- 보너스 번호 입력 받기
  - 당첨 번호와 중복되지 않음 검증
- 당첨 판정 및 통계 계산
  - 1~5등 판정 로직 구현 (보너스 볼 포함 조건 포함)
  - 등수별 개수 집계
  - 수익률 계산 및 소수점 둘째 자리에서 반올림하여 출력(예: 62.5%)
- 예외 처리
  - 잘못된 입력에 대해 명확한 예외 타입 사용(예: IllegalArgumentException)
  - 예외 메시지는 "[ERROR]"로 시작
  - UI에서 예외 발생 시 메시지 출력 후 해당 입력부터 재요청
- 출력 포맷
  - 발행 수량 출력: "N개를 구매했습니다."
  - 각 로또 번호는 한 줄에 하나씩 출력 (오름차순)
  - 당첨 통계 형식은 과제 예시와 동일하게 출력
- 단위 테스트
  - 비즈니스 로직(로또 생성기, 당첨 판정기, 통계 계산기, 수익률 계산)은 JUnit 5 + AssertJ로 테스트
  - UI(System.in/System.out)는 테스트 대상에서 제외
- 기타 요구사항 준수
  - JDK 21, `Application.main()` 진입점
  - indent depth 최대 2, 메서드 길이 15줄 이하, `else`/삼항 연산자 사용 금지
  - Java `enum` 활용
  - `Lotto` 제공 클래스 사용 (필드 변경 불가)
